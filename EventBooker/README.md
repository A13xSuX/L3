# EventBooker

EventBooker — сервис бронирования мероприятий с автоматической отменой неоплаченных броней по истечении времени.

## Что реализовано

Сервис позволяет:

- создавать мероприятия;
- бронировать места;
- подтверждать бронь;
- автоматически отменять неподтвержденные брони через заданный интервал времени;
- просматривать информацию о мероприятии, свободных местах и текущих бронях.

Для отмены просроченных броней используется фоновый обработчик (`cron-like scheduler`), который периодически проверяет брони со статусом `pending` и истекшим временем оплаты.

Все операции с бронированием и подтверждением выполняются с использованием транзакций, чтобы избежать гонок данных.

---
## API

### Создание мероприятия
`POST /events`

Создает новое мероприятие.

### Бронирование места
`POST /events/{id}/book`

Создает бронь на мероприятие.

### Подтверждение брони
`POST /events/{id}/confirm`

Подтверждает бронь, если мероприятие требует оплаты.

> В текущей реализации параметр `{id}` в этом маршруте используется как `bookingID`.

### Получение информации о мероприятии
`GET /events/{id}`

Возвращает информацию о мероприятии, список броней и количество свободных мест.

---

## Логика бронирования

При бронировании:

- если мероприятие не требует оплаты, бронь сразу получает статус `confirmed`;
- если мероприятие требует оплаты, бронь создается со статусом `pending`;
- для такой брони выставляется время истечения (`expired_at`);
- если бронь не подтверждена вовремя, фоновый обработчик переводит ее в статус `canceled` и возвращает место обратно.

---

## Фоновый обработчик

В проекте реализован фоновый worker, который:

- запускается вместе с приложением;
- через заданный интервал времени проверяет истекшие брони;
- отменяет брони со статусом `pending`, у которых истек `expired_at`;
- возвращает свободное место в мероприятие.

## Веб-интерфейс

Реализован простой веб-интерфейс с двумя страницами:

- пользовательская часть;
- административная часть.

### Пользовательская часть
Позволяет:

- просматривать список мероприятий;
- открывать детали мероприятия;
- бронировать место;
- подтверждать бронь;
- наблюдать, как неподтвержденные брони автоматически отменяются.

### Административная часть
Позволяет:

- создавать мероприятия;
- просматривать список мероприятий;
- смотреть детали мероприятия и список броней.

---

## Структура веб-части

Файлы расположены в папке `web/`:

- `index.html` — главная страница;
- `user.html` — пользовательская часть;
- `admin.html` — административная часть;
- `styles.css` — стили;
- `app.js` — логика фронтенда.

## Запуск проекта

### 1. Поднять базу данных
Нужно поднять PostgreSQL и указать корректные параметры подключения в конфиге.

### 2. Применить миграции
Применить миграции через `goose`.

### 3. Запустить приложение

Из папки `cmd`:

```bash
go run .\main.go
```
После запуска сервис будет доступен на:

```http://localhost:8080```
